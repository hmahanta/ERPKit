# Authors

ERPKit is built and maintained by its community. This file recognizes the
people who have created, led, and contributed to the project.

For contribution guidelines, see [`CONTRIBUTING.md`](CONTRIBUTING.md). For
how these roles are defined and how they are earned, see
[`GOVERNANCE.md`](GOVERNANCE.md).

## Founder

ERPKit was created to address a recurring gap identified across enterprise
ERP integration work: the absence of a shared, disciplined Python library
for metadata-driven validation, financial-grade business rules, transaction
integrity, and auditability (see the ERPKit architecture whitepaper,
Sections 1–3, for the full motivation). The Founder authored the initial
core architecture — the Validator, MetaFlow, Transaction Manager, and
Approval Engine modules — and continues to serve as a Maintainer under the
governance model described in `GOVERNANCE.md`.

### How to Be Added

Contributors are added to this file as part of the pull request that makes
their first contribution — add yourself in the same pull request, in the
format:

```
- @<github-username> — <display name, optional> — <short description of contribution area>
```

Maintainers will not remove a contributor entry once added, consistent with
recognizing all contributions permanently regardless of whether the
contributor remains active.
